# نظام تسجيل الدخول - Pub's Pizza

## ✅ ما تم إضافته

### 1. **AuthContext.jsx** - نظام إدارة المصادقة
- إدارة حالة المستخدم الحالي
- دوال تسجيل دخول منفصلة للمسؤول والمستخدم
- تخزين بيانات المستخدم في localStorage
- التحقق من صلاحيات المسؤول

### 2. **صفحة تسجيل دخول المسؤول** (`AdminLoginPage.jsx`)
**البيانات التجريبية:**
- البريد: `admin@pubspizza.com`
- كلمة المرور: `admin123`

**الميزات:**
- واجهة احترافية باللغة العربية
- إظهار/إخفاء كلمة المرور
- زر تسجيل دخول تجريبي سريع
- معالجة الأخطاء

### 3. **صفحة تسجيل دخول المستخدم** (`UserLoginPage.jsx`)
**بيانات مطلوبة:**
- الاسم الكامل (حد أدنى حرفين)
- رقم الهاتف (حد أدنى 10 أرقام)
- الموافقة على الشروط والأحكام

**الميزات:**
- لا تحتاج إلى كلمة مرور
- تحقق من صحة المدخلات
- واجهة صديقة المستخدم

### 4. **مكون الحماية** (`ProtectedRoute.jsx`)
- يحمي صفحة `/admin` من الدخول غير المصرح
- إعادة توجيه تلقائية إلى صفحة تسجيل دخول المسؤول

### 5. **تحديثات الـ Navbar**
**ميزات جديدة:**
- عرض المستخدم الحالي
- قائمة منسدلة للمستخدم تعرض معلوماته
- زر تسجيل الخروج
- رابط الدخول للمستخدمين غير المسجلين
- ظهور رابط الإدارة فقط للمسؤولين

### 6. **تحديثات التوجيه** (`App.jsx`)
**مسارات جديدة:**
- `/login` - صفحة تسجيل دخول المستخدم
- `/admin-login` - صفحة تسجيل دخول المسؤول
- `/admin` - مستحفظة بحماية (ProtectedRoute)

---

## 🔐 تدفق المصادقة

### تسجيل دخول المسؤول:
```
1. المستخدم يزور /admin-login
2. يدخل البيانات (admin@pubspizza.com / admin123)
3. يتم التحقق من الصحة
4. إعادة توجيه إلى /admin
ProtectedRoute تتحقق من صلاحيات المسؤول
```

### تسجيل دخول المستخدم:
```
1. المستخدم يزور /login
2. يدخل اسمه ورقم هاتفه
3. يوافق على الشروط والأحكام
4. يتم التحقق من الصحة
5. إعادة توجيه إلى /menu
```

---

## 📱 البيانات المحفوظة

تتم مزامنة البيانات التالية مع `localStorage`:
- معلومات المستخدم الحالي
- الدور (مسؤول / مستخدم عادي)
- وقت تسجيل الدخول

---

## 🔄 الخروج (Logout)

- الزر موجود في قائمة المستخدم في الـ Navbar
- يحذف المستخدم الحالي من localStorage
- يعيد التوجيه إلى الرئيسية

---

## 📝 ملاحظات أمان

**للإنتاج:**
- لا تخزن كلمات المرور في الـ localStorage
- استخدم Firebase Authentication للمسؤولين
- أضف تشفير للبيانات الحساسة
- استخدم HTTPS فقط
- أضف حد أقصى لمحاولات تسجيل الدخول

---

## 🚀 اختبار سريع

1. انقر على زر الدخول في Navbar
2. جرب تسجيل دخول مستخدم عادي
3. لاحظ ظهور المستخدم في Navbar
4. انقر على زر "هل أنت مسؤول"
5. جرب البيانات التجريبية (`admin@pubspizza.com` / `admin123`)
6. يجب أن تظهر صفحة الإدارة
7. انقر على قائمة المستخدم واختر "تسجيل خروج"
